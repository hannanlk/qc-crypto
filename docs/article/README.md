# I Ran Shor's Algorithm on a Real Quantum Computer. RSA Survived — Here's Why.

### A hands-on, honest tour of how quantum computing actually changes modern cryptography — Shor, Grover, and the post-quantum replacement — with runnable code for every claim.

---

Quantum computing "breaks encryption" is one of the most repeated and least
understood claims in tech. It is neither the apocalypse the headlines imply nor the
non-event the skeptics claim. The truth is specific: quantum computers break *certain
mathematical problems*, and which cryptography that endangers depends entirely on
*which* problem it rests on.

This series builds that understanding from the ground up, in code you can run. We
factor a toy RSA modulus with Shor's algorithm on a simulator and on real IBM Quantum
hardware; we show why Grover's algorithm only dents AES instead of shattering it; and
we end at ML-KEM, the NIST-standardised replacement for RSA. Every claim is backed by a
runnable demonstration, every quantum result is compared against a classical baseline,
and — most importantly — the line between *what runs today* and *what needs a future
fault-tolerant machine* is drawn explicitly and never blurred.

**Audience:** software engineers, cybersecurity professionals, and CS students. No prior
quantum computing background is assumed; the mathematics is built up as needed.

4. [**ML-KEM: The Post-Quantum Replacement**](04-mlkem.md) — lattice-based key
   encapsulation (FIPS 203), and why it resists all *known* quantum attacks.

## The one-paragraph version

Shor's *exponential* speedup breaks the factoring and discrete-log problems under RSA and
ECDH — so those must be replaced. Grover's *quadratic* speedup only halves symmetric key
strength — so AES survives a key-size bump to 256 bits. And ML-KEM replaces the broken
asymmetric primitives with lattice-based ones. Neither apocalypse nor non-event: a
specific, addressable migration, already underway because of *harvest-now, decrypt-later*.

## Running the code

See the [project README](../../README.md). In short:

```bash
python -m pip install -e ".[dev,quantum,pqc]"
qcrypto rsa-demo        # Part 0
qcrypto shor-demo       # Part 1 (Part 2 adds --backend ibm)
qcrypto grover-demo     # Part 3
qcrypto mlkem-demo      # Part 4
```
